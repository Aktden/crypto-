
#  Выводы

## Результаты работы

В ходе выполнения проекта было разработано демонстрационное приложение на C++ с использованием библиотеки **Crypto++**, реализующее следующие криптографические механизмы:

-  Хеширование строк алгоритмами SHA-256, SHA-1, MD5
-  Аутентификация сообщений через HMAC-SHA256
-  Симметричное шифрование AES-128 в режиме CBC
-  Деривация ключа из пароля через PBKDF2-HMAC-SHA256
-  Безопасное хранение ключей через `SecByteBlock`
-  Замер производительности криптографических операций



## Выводы по библиотеке Crypto++

**Достоинства:**
- Огромное количество реализованных алгоритмов в одной библиотеке
- Нативный C++ API без C-прослоек
- `SecByteBlock` автоматически обнуляет память — защита от утечки ключей
- Гибкая архитектура пайплайнов позволяет комбинировать операции
- Активно поддерживается и обновляется

**Недостатки:**
- Синтаксис пайплайнов непривычен для новичков
- Документация менее подробна по сравнению с OpenSSL
- Отсутствие сертификации FIPS 140-2



## Выводы по алгоритмам

| Алгоритм | Вывод |
|---|---|
| SHA-256 | Рекомендуется для всех новых проектов |
| SHA-1 | Использовать только для обратной совместимости |
| MD5 | Использовать только для некриптографических задач (контрольные суммы) |
| HMAC-SHA256 | Надёжный способ аутентификации сообщений |
| AES-128 CBC | Достаточен для обучения; в продакшне предпочтительнее AES-256-GCM |
| PBKDF2 | Правильный подход к хранению паролей; рекомендуется ≥ 100 000 итераций |



## Направления для дальнейшего развития

1. Заменить AES-CBC на **AES-GCM** для аутентифицированного шифрования
2. Добавить **ECDSA** и **ECDH** для асимметричной криптографии
3. Реализовать шифрование **файлов** с использованием `FileSource` / `FileSink`
4. Увеличить количество итераций PBKDF2 до **100 000+**
5. Добавить **ChaCha20-Poly1305** как альтернативу AES для мобильных устройств
6. Написать **unit-тесты** для каждого криптографического модуля
