#  Crypto++ Demo - C++ Криптографическая Демонстрация

Проект, демонстрирующий основные возможности библиотеки **Crypto++** на C++.
Покрывает хеширование, HMAC-аутентификацию, симметричное шифрование AES и деривацию ключей из пароля.


##  Структура проекта
```
crypto_demo/
├── include/
│   ├── aes_demo.h       # Заголовок класса AESDemo
│   ├── hash_demo.h      # Заголовок класса HashDemo
│   └── utils.h          # Утилиты, таймер, вывод
├── src/
│   ├── main.cpp         # Точка входа
│   ├── aes_demo.cpp     # Реализация AES-шифрования
│   ├── hash_demo.cpp    # Реализация хеш-функций
│   └── utils.cpp        # Реализация утилит
├── CMakeLists.txt       # Сборка через CMake
└── crypto_demo.vcxproj  # Проект Visual Studio
```

##  Сборка и запуск

### Linux / macOS

**Установка Crypto++:**
```bash
# Ubuntu / Debian
sudo apt-get install libcrypto++-dev

# macOS (Homebrew)
brew install cryptopp
```

**Сборка через CMake:**
```bash
git clone https://github.com/ВАШ_ЮЗЕР/crypto_demo.git
cd crypto_demo
mkdir build && cd build
cmake ..
make
./crypto_demo
```

### Windows (Visual Studio)

1. Установить Crypto++ через [vcpkg](https://github.com/microsoft/vcpkg):
```bash
vcpkg install cryptopp:x64-windows
```
2. Открыть `crypto_demo.vcxproj` в Visual Studio
3. Собрать проект (`Ctrl+Shift+B`)
4. Запустить (`F5`)

##  Что демонстрирует программа

###  Модуль 1 — Хеш-функции (`HashDemo`)

Вычисление дайджестов одного сообщения тремя алгоритмами:

| Алгоритм | Размер | Статус |
|---|---|---|
| SHA-256 | 32 байта |  Безопасен |
| SHA-1 | 20 байт |  Устарел |
| MD5 | 16 байт |  Не для безопасности |

**Пример вывода:**
```
 Хеширование сообщений
------------------------------
    Исходное сообщение: "Hello, Crypto++!"

  SHA-256: 9BDE5CCB...
  Длина: 32 байта

  SHA-1:   A94A8FE5...
  Длина: 20 байт

  MD5:     8B1A9953...
  Длина: 16 байт
```

---

###  Модуль 2 — HMAC-аутентификация (`HashDemo`)

Вычисление кода аутентификации сообщения с секретным ключом по алгоритму **HMAC-SHA256**.

**Пример вывода:**
```
 HMAC-SHA256 Аутентификация
------------------------------
    Сообщение: "Важное сообщение"
    Ключ: "секретный_ключ"
  HMAC: 3A7F2C1D...
   Аутентификация успешна
```

---

###  Модуль 3 — AES-128 CBC шифрование (`AESDemo`)

Генерация случайного ключа и IV, шифрование и дешифрование сообщения в режиме CBC.

**Пример вывода:**
```
 AES-128 CBC Шифрование
------------------------------
    Исходный текст: "Секретное сообщение для AES!"
    Ключ: A1B2C3D4...
    IV:   F1E2D3C4...
    AES шифрование: 0.031 ms
  Зашифровано (hex):    4F3A1C...
  Зашифровано (base64): TzocA...
  Расшифровано: Секретное сообщение для AES!
   Шифрование и расшифрование успешно!
```

---

###  Модуль 4 — Шифрование с паролем PBKDF2 (`AESDemo`)

Деривация ключа AES из пароля пользователя с помощью **PBKDF2-HMAC-SHA256** (10 000 итераций) и случайной соли.

**Пример вывода:**
```
 Шифрование с паролем (PBKDF2)
------------------------------
    Исходный текст: "Данные, защищённые паролем"
    Пароль: MyPassword123
    Соль: 8F3A2C1D...
    Производный ключ: 5E7B9A3F...
  Зашифровано: 2C4F8A...
   Шифрование с паролем выполнено
```

##  Ключевые классы Crypto++, использованные в проекте

| Класс | Заголовок | Назначение |
|---|---|---|
| `CryptoPP::SHA256` | `<cryptopp/sha.h>` | Хеш SHA-256 |
| `CryptoPP::SHA1` | `<cryptopp/sha.h>` | Хеш SHA-1 |
| `CryptoPP::MD5` | `<cryptopp/md5.h>` | Хеш MD5 |
| `CryptoPP::HMAC<SHA256>` | `<cryptopp/hmac.h>` | HMAC-аутентификация |
| `CryptoPP::CBC_Mode<AES>` | `<cryptopp/modes.h>` | AES в режиме CBC |
| `CryptoPP::PKCS5_PBKDF2_HMAC` | `<cryptopp/pwdbased.h>` | Деривация ключа из пароля |
| `CryptoPP::AutoSeededRandomPool` | `<cryptopp/osrng.h>` | Генератор случайных чисел |
| `CryptoPP::SecByteBlock` | `<cryptopp/secblock.h>` | Безопасный буфер для ключей |
| `CryptoPP::HexEncoder` | `<cryptopp/hex.h>` | Кодирование в HEX |
| `CryptoPP::Base64Encoder` | `<cryptopp/base64.h>` | Кодирование в Base64 |

##  Важные замечания по безопасности

> **CBC без аутентификации** — в данном проекте AES-CBC используется в учебных целях. В продакшен-коде используйте **AES-GCM**, который совмещает шифрование и аутентификацию.

> **MD5 и SHA-1** — продемонстрированы исключительно в образовательных целях. Не используйте их для криптографических задач.

> **SecByteBlock** — все ключи хранятся в `SecByteBlock`, который автоматически обнуляет память при уничтожении объекта.
